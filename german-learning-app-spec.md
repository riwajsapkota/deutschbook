# Deutschbook — Personal German Learning App

## Product Spec v1.0

---

## Vision

A personal, living German grammar workbook that grows with every lesson. You dump in raw materials from your teacher — PDFs, audio, Excel files, notes — and an AI agent organizes everything into a structured, topic-based exercise book. The app doesn't just store your learning; it teaches you, quizzes you, and makes sure you retain what you've learned through spaced repetition.

Think of it as a textbook that writes itself from your lessons, and then tutors you from its own pages.

---

## Core Principles

1. **Minimal effort to input, maximum structure on output.** You should never manually tag, categorize, or organize. The agent handles that.
2. **Topics, not dates.** The organizing unit is a grammar concept (e.g., Konjunktiv II), not "Lecture 8." Multiple sessions feed into the same topic chapter.
3. **Retention through repetition.** Content isn't archived — it's actively resurfaced for review, practice, and quizzing.
4. **Living document.** The book grows, improves, and reorganizes itself over time as more material is added.

---

## User Profile

- Single user (you)
- B1/B2 German learner
- Twice-weekly sessions with a private teacher
- Receives materials via email: PDFs (exercises), audio files, Excel sheets, reference book recommendations
- Has existing notes in Apple Notes
- Goal: retain grammar through structured repetition

## Architecture

### Tech Stack (Local-First)

| Layer | Technology | Notes |
|-------|-----------|-------|
| Frontend | Next.js (React) | Runs locally via `npm run dev` |
| Backend | Next.js API Routes | Server-side logic, agent orchestration |
| Database | SQLite | Single file, zero config, portable |

| File Storage | Local filesystem | `/data/uploads/` for raw files, `/data/processed/` for converted content |

When ready to deploy:
- Push code to GitHub
- Deploy frontend + API to Vercel
- Swap SQLite → Postgres (e.g., Supabase)
- Swap local file storage → Supabase Storage or S3
- Add authentication (Supabase Auth or NextAuth)
- Same codebase, minimal changes

## Data Model

### Session

A record of a single lesson with your teacher.

```
Session {
  id: string
  date: date
  lecture_number: integer (optional)
  raw_notes: text (optional, free-form notes you jot down)
  status: "inbox" | "processed" | "partially_processed"
  created_at: datetime
}
```

### Chapter (Topic)

The core organizing unit. Each chapter covers one grammar concept or vocabulary theme.

```
Chapter {
  id: string
  title: string                    // e.g., "Konjunktiv II"
  level: "B1" | "B2" | "mixed"
  category: "grammar" | "vocabulary" | "other"
  summary: text                    // AI-generated explanation of the topic
  theory: text                     // Teaching content — rules, explanations, examples
  created_at: datetime
  updated_at: datetime
}
```

### Exercise

An interactive exercise, converted from a PDF or generated by the agent.

```
Exercise {
  id: string
  chapter_id: string              // belongs to a Chapter
  session_id: string              // originated from a Session
  source_file: string             // path to original PDF/Excel
  type: "fill_in_blank" | "multiple_choice" | "reorder" | "translate" | "free_response"
  instruction: text               // e.g., "Ergänzen Sie das Verb im Konjunktiv II"
  items: [                        // individual questions within the exercise
    {
      id: string
      prompt: text                // the sentence/question with blanks
      blanks: [                   // for fill-in-blank type
        { position: integer, correct_answer: string, alternatives: [string] }
      ]
      options: [string]           // for multiple choice
      correct_answer: string      // for simple answer types
      explanation: text           // why this is the answer
    }
  ]
  difficulty: "easy" | "medium" | "hard"
  created_at: datetime
}
```

### Vocabulary

Individual vocabulary items, tied to chapters.

```
VocabularyItem {
  id: string
  chapter_id: string (optional)   // can belong to a grammar chapter for context
  word: string                    // German word
  article: string (optional)      // der/die/das for nouns
  plural: string (optional)
  translation: string             // English or native language
  example_sentence: string
  part_of_speech: string          // noun, verb, adjective, etc.
  tags: [string]                  // additional categorization
  created_at: datetime
}
```

### Attempt

Tracks every time you do an exercise or vocabulary quiz.

```
Attempt {
  id: string
  exercise_id: string (optional)
  vocabulary_item_id: string (optional)
  score: float                    // 0.0 to 1.0
  answers: [                     // what you actually answered
    { item_id: string, given_answer: string, correct: boolean }
  ]
  self_assessment: "solid" | "shaky" | "forgotten" (optional)
  attempted_at: datetime
}
```

### Material

Raw uploaded files linked to sessions.

```
Material {
  id: string
  session_id: string
  file_path: string
  file_type: "pdf" | "audio" | "excel" | "image" | "text"
  original_filename: string
  processing_status: "pending" | "processed" | "failed"
  uploaded_at: datetime
}
```

### ReviewSchedule (Spaced Repetition)

Tracks when each chapter/exercise/vocab item is due for review.

```
ReviewSchedule {
  id: string
  target_type: "chapter" | "exercise" | "vocabulary"
  target_id: string
  next_review_at: datetime
  interval_days: integer          // current spacing interval
  ease_factor: float              // how easy/hard (adjusts interval)
  review_count: integer
}
```

## Features

### 1. Session Inbox

**Purpose:** Quick, low-friction capture after each lesson.

**How it works:**
- You create a new session entry ("Lecture 9 — May 25, 2026")
- Upload files: drag-and-drop PDFs, Excel files, audio
- Optionally paste free-form notes ("We practiced Relativsätze, also reviewed Passiv from last time")
- Hit "Process" — the agent takes over

**UI:** Simple form. Date picker, file upload zone, text area for notes, one big button to trigger processing.

### 2. AI Agent Processing

**Purpose:** Automatically organize raw session materials into the structured book.

**What the agent does when you hit "Process":**

1. **Reads uploaded PDFs/Excel** — extracts text, identifies exercise structures
2. **Identifies grammar topics** — "This PDF contains fill-in-the-blank exercises for Konjunktiv II and Passiv"
3. **Maps to existing chapters** — if a Konjunktiv II chapter exists, append there; if not, create one
4. **Converts exercises** — transforms PDF exercises into interactive format (fill-in-blank fields, answer keys)
5. **Extracts vocabulary** — pulls out key words, generates definitions and example sentences
6. **Generates/updates theory** — if the chapter lacks an explanation, the agent writes one; if new material adds nuance, it updates the existing explanation
7. **Cross-references** — notes that "Lecture 9 covered similar material to Lecture 3" and links them
8. **Sets up review schedule** — newly added content gets scheduled for its first review

**Agent prompt strategy:**
- Use Claude API with structured output (JSON) for exercise conversion
- Multi-step: first classify the content, then convert exercises, then generate theory
- Include existing chapter content in context so the agent knows what already exists

### 3. The Book (Chapter View)

**Purpose:** Your personal, growing grammar reference and workbook.

**Chapter list view:**
- All chapters listed, grouped by category (grammar vs. vocabulary)
- Each shows: title, level, number of exercises, last practiced date, retention indicator
- Retention indicator: green (solid), yellow (getting stale), red (overdue for review)
- Search and filter by level, category, retention status

**Single chapter view:**
- **Theory section** — the explanation, rules, tables, examples. Readable, clean, textbook-like formatting.
- **Exercises section** — list of interactive exercises, each showing source session and completion status
- **Vocabulary section** — words tied to this topic
- **Session history** — "Covered in: Lecture 1 (Jan 5), Lecture 8 (Mar 12), Lecture 9 (May 25)"
- **Retention stats** — accuracy over time, last review date, next scheduled review

### 4. Interactive Exercises

**Purpose:** Practice grammar digitally, with immediate feedback.

**Exercise experience:**
- Present the exercise with interactive blanks (text inputs where you type answers)
- Submit when done
- Immediate feedback: correct answers highlighted green, wrong answers highlighted red with the correct answer shown
- For wrong answers, show a brief explanation of why
- Option to retry just the ones you got wrong
- After completing, self-assess: "How did that feel? Solid / Shaky / Forgotten"
- Results saved as an Attempt, feeding the spaced repetition system

**Exercise types to support:**
- **Fill in the blank** — most common from teacher's PDFs
- **Multiple choice** — choose the correct option
- **Sentence reordering** — drag words into correct order
- **Translation** — translate a sentence, AI evaluates your answer
- **Free response** — write a sentence using a grammar structure, AI gives feedback

### 5. Quiz Mode

**Purpose:** Active recall and retention testing across topics.

**How it works:**
- The app selects items due for review (based on spaced repetition schedule)
- Mixes exercise types and topics for variety
- Can also be filtered: "Quiz me only on Konjunktiv II" or "Quiz me on all B1 grammar"
- Includes both grammar exercises and vocabulary
- After each item, feedback + explanation
- Session summary at the end: "You reviewed 15 items. 11 correct. Konjunktiv II is solid. Relativsätze needs work."

**Spaced repetition logic:**
- Uses SM-2 algorithm (same as Anki) adapted for exercises
- Correct + "solid" → increase interval (show again later)
- Correct + "shaky" → maintain interval
- Incorrect or "forgotten" → reset interval (show again soon)

### 6. Teach Mode

**Purpose:** Let the app explain a concept to you before you practice it.

**How it works:**
- Open a chapter → "Teach me this"
- The app presents the theory section in a structured, step-by-step way
- Can include: rules, conjugation/declension tables, example sentences, common mistakes
- At the end of the teaching sequence, offers: "Ready to practice?" → launches exercises for that chapter
- If you have questions, you can ask the AI inline and it responds in context of that chapter's content

### 7. Apple Notes Import

**Purpose:** Bring in your existing learning notes so the book starts with a foundation.

**How it works:**
- Export Apple Notes as text/HTML (Apple Notes supports this)
- Upload to the app
- Agent processes them the same way as session materials: identifies topics, extracts content, files into chapters
- Your old notes become part of the book's theory and reference sections

### 8. Dashboard

**Purpose:** At-a-glance view of your learning state.

**Shows:**
- **Due for review** — chapters and exercises that need attention today
- **Recent sessions** — last few lessons with processing status
- **Chapter health** — a grid or list of all topics with retention indicators
- **Stats** — total exercises completed, vocabulary learned, streak, accuracy trends
- **Quick actions** — "Start quiz," "Add new session," "Review weakest topic"

## User Flows

### Flow 1: After a Lesson

```
1. Open app → Dashboard
2. Click "New Session"
3. Enter date, lecture number
4. Upload PDF(s) from teacher
5. Paste any quick notes
6. Click "Process"
7. Agent runs (may take 30-60 seconds)
8. Session moves to "processed"
9. New/updated chapters appear in the book
```

### Flow 2: Daily Review (5-10 minutes)

```
1. Open app on phone → Dashboard
2. See "7 items due for review"
3. Tap "Start Quiz"
4. Work through mixed exercises and vocabulary
5. Get feedback after each item
6. See summary at end
7. Done — retention maintained
```

### Flow 3: Deep Study

```
1. Open app on PC → Book
2. Browse chapters, find "Relativsätze" marked yellow (getting stale)
3. Open chapter → read Theory section to refresh
4. Tap "Practice" → work through exercises
5. Try the harder ones, check answers
6. Chapter goes green
```

### Flow 4: Reviewing Past Material

```
1. Open app → Book
2. Search "Passiv"
3. See chapter with material from Lecture 2, 5, and 9
4. All exercises visible, with your past scores
5. Redo the ones you previously struggled with
```

## Mobile Considerations

Since this is a responsive web app (not a native app), mobile optimization focuses on:

- **Dashboard** — fully usable, shows due reviews prominently
- **Quiz mode** — optimized for phone, large tap targets, one item at a time
- **Chapter reading** — scrollable, readable theory sections
- **Exercise input** — text inputs sized for mobile keyboards
- **Session creation** — may be less common on mobile, but should work (file upload via phone camera for paper materials)
- **No heavy editing on mobile** — save that for PC

## AI Agent Technical Details

### API Usage

All AI processing uses the Claude API (Anthropic). You'll need an API key stored in a local `.env` file.

### Agent Tasks & Prompts

**Task 1: Content Classification**
- Input: Extracted text from PDF/Excel
- Output: JSON with identified topics, exercise types, vocabulary
- Prompt strategy: Provide the text + list of existing chapter titles → agent maps content to existing or new chapters

**Task 2: Exercise Conversion**
- Input: Raw exercise text (e.g., "Ergänzen Sie: Wenn ich Zeit ____ (haben)...")
- Output: Structured exercise JSON with prompts, blanks, and correct answers
- Prompt strategy: Ask for structured JSON output with the exercise items, correct answers, and brief explanations

**Task 3: Theory Generation**
- Input: Exercises + notes for a topic, plus any existing theory
- Output: Updated theory text — clear explanation of the grammar rule with examples
- Prompt strategy: "You are writing a chapter of a personal grammar textbook for a B1/B2 learner. Based on these exercises and notes, write or update the explanation for [topic]."

**Task 4: Vocabulary Extraction**
- Input: Session materials
- Output: List of vocabulary items with translations, articles, example sentences
- Prompt strategy: Extract significant vocabulary, focusing on B1/B2 level words

**Task 5: Quiz Evaluation (for free-response)**
- Input: The grammar rule, the prompt, the user's answer
- Output: Correctness assessment, specific feedback, corrected version if needed
- Prompt strategy: Evaluate as a German teacher would — check grammar, word order, case, etc.

### Processing Pipeline

```
Upload files
    ↓
Extract text (pdf-parse for PDFs, xlsx library for Excel)
    ↓
Task 1: Classify content → identify topics + exercise types
    ↓
Task 2: Convert exercises → structured interactive format
    ↓
Task 3: Generate/update theory for each identified topic
    ↓
Task 4: Extract vocabulary
    ↓
Save everything to SQLite
    ↓
Update review schedules for new content
    ↓
Mark session as "processed"
```
